## 1. 根本原因：日本語 ↔ 英語 の「言語ギャップ」

OpenAlex の分類モデルは多言語対応を謳っていますが、ソースコードの精査により以下の構造的問題が確認されました。

- **学習データが英語中心**（欧米論文 430 万件、日本語の比率が極めて少ない）
- **引用ネットワークへの依存**（国際引用が少ない日本語論文では機能しない）
- **モデルの英語偏重ファインチューニング**により日本語処理能力が失われている

これらが「Military Technology and Strategies」への大量誤分類の直接的な原因です。

## 3. 新たに提案する 2 つのアプローチ

### 提案A：機械翻訳前処理（最大インパクト）

日本語論文の要旨・タイトルを**英語に翻訳してから**埋め込み検索を行う手法です。

```
【現在】 日本語テキスト → 多言語埋め込み → 英語Topic説明文との照合
                              ↑ここに言語ギャップが残る

【提案A】 日本語テキスト → 英語翻訳 → 英語テキスト → 英語特化モデルで照合
                                                     ↑英語同士で精密照合
```

- 翻訳は **OPUS-MT**（ヘルシンキ大学提供の無料ローカルモデル）を使用
- 追加費用は発生しない
- 言語ギャップを根本から解消するため、精度向上効果が最も大きいと見込まれます

### 提案B：階層的分類（追加コストなし）

OpenAlex の Topic は **7 分野（Domain）→ 34 領域（Field）→ 4,521 Topic** の階層構造を持っています。現在は 4,521 件を一度に検索していますが、先に分野を絞ってから Topic を探す階層的アプローチを導入します。

```
【現在】 論文 → 4,521 Topic から一括検索 → 誤った分野に引き寄せられる

【提案B】 論文 → まず 7 分野を判定（Health Sciences など）
               → その分野内の約 200 Topic のみで検索
               → 医療論文が軍事分野に誤分類されることを構造的に防止
```

- 既存のモデル・インデックスをそのまま流用できる
- 実装コストがほぼゼロで、広域誤分類を大幅に削減できます

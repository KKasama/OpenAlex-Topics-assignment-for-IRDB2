## 翻訳×埋め込みモデル比較実験（v4）の結果

長野様からご提案いただいた SciBERT および高精度翻訳モデルの組み合わせについて、以下の 4 通りを検証しました。1,000 件のサンプルで Topic 付与を実行し、そのうち事前に正解ラベルを作成済みの 97 件を用いて精度評価を行いました。

なお、PLaMo-2-translate（9.5B パラメータ）は実行に 32GB 以上のメモリが必要なため、手元の Mac mini M4（RAM 16GB）では動作させることができませんでした。そのため、同じく日本語特化の翻訳モデルである **FuguMT（約 300MB）** を代替として使用しました。Gemini API については、前回の失敗原因（ボイラープレート要旨の混入・タイトルと要旨の一括翻訳）を修正した改訂版を使用しています。

| 手法 | 翻訳 | 埋め込み | Primary 一致率 | Top-3 Hit 率 |
|---|---|---|---|---|
| v1（ベースライン） | なし | e5-base | 20.6% | 34.0% |
| v2（現行推奨） | なし | e5-base + BM25 | 22.7% | 35.1% |
| v4-FuguMT + e5 | FuguMT | e5-base | 16.5% | 26.8% |
| **v4-Gemini + e5（修正版）** | Gemini | e5-base | **22.7%** | **37.1%** ← 最高 |
| v4-FuguMT + SciBERT | FuguMT | SciBERT | 5.2% | 6.2% |
| v4-Gemini + SciBERT | Gemini | SciBERT | 7.2% | 9.3% |

## 主な知見

**① Gemini 翻訳（修正版）＋ e5-base が Top-3 Hit 率で初めて v2 を上回りました**

ボイラープレートの除去とタイトル・要旨の個別翻訳により、Top-3 Hit 率が **37.1%**（v2 の 35.1% から +2 ポイント）に向上しました。「候補 3 件の中に正解が含まれる」確率の向上は、実際のレコメンデーション品質に直結するため、意義のある改善と考えております。

**② SciBERT は本タスクに不適合**

Primary 一致率が 5〜7% まで低下しました。SciBERT は英語論文のみで学習されており、翻訳後テキストと OpenAlex Topic 説明文の類似度計算がうまく機能しないことが原因と考えられます。

**③ FuguMT の翻訳精度は不十分**

FuguMT は OPUS-MT より改善されているものの、multilingual-e5-base が内包する日英クロスリンガル能力を上回るほどの翻訳品質には至りませんでした。PLaMo-2-translate であれば異なる結果になる可能性は残りますが、現時点では確認できておりません。
